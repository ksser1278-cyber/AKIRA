# [별첨 91. 클로드 코드 소스 코드 분석서 - 클로드 코드 가이드]

**분석 대상:** `claude code source code (2026-03-31)`  
**분석 일자:** 2026-04-01  
**총 소스 파일:** 약 1,884개 (TypeScript + React)

Anthropic의 Claude Code 전체 소스 코드가 노출되었습니다. 본 장에서는 해당 소스 코드에 대한 심층 분석 내용을 공유합니다. 다만, 소스 코드 분석서임에도 불구하고 유출된 소스 코드는 단 한 줄도 직접 공개하지 않습니다. 대신 전체 구조, 기능, 동작 흐름을 중심으로 분석을 진행했습니다. 소스 코드 분석서 공유에 대한 공정성 논란의 여지가 있을 수 있으나, Claude Code를 보다 깊이 이해하고 효과적으로 활용하는 데 도움이 될 것이라는 판단 하에 공유드립니다.

## 요약: 세 문장으로 이해하기
Claude Code는 사용자의 자연어 요청을 받아 AI가 어떤 도구를 써야 하는지 판단하고, 권한 확인을 거쳐 안전하게 실행한 후, 결과를 다시 AI에게 돌려주는 루프를 핵심으로 한다. 이 루프를 감싸는 시스템들(인증, 설정, 상태 관리, MCP, 플러그인, 브리지, 코디네이터)이 다양한 환경과 사용 사례를 지원한다. 모든 설계 결정은 안전성(위험한 작업 차단), 성능(스트리밍, 병렬 실행, 캐싱), 확장성(도구, 스킬, 플러그인, MCP)의 세 가지 원칙을 따른다.

## 1. Claude Code란 무엇인가
Claude Code는 Anthropic이 만든 공식 CLI(터미널 명령줄) 도구다. 사용자가 터미널에서 Claude AI와 대화하면서 코드를 읽고, 수정하고, 실행할 수 있게 해준다. 단순한 챗봇이 아니라, 파일을 직접 편집하고, 셸 명령을 실행하고, 웹을 검색하고, 외부 서비스와 연동하는 "AI 소프트웨어 엔지니어"에 가깝다. 

기술적으로는 TypeScript로 작성되었고, 터미널 UI는 React 기반의 TUI(Terminal User Interface) 프레임워크인 Ink를 사용한다. 상태 관리는 Zustand 라이브러리를 채택했으며, 빌드 시스템은 bun을 사용해서 사용하지 않는 코드를 자동으로 제거(tree-shaking)한다.

## 2. 전체 구조를 한눈에 보기
Claude Code의 동작은 크게 네 단계로 요약할 수 있다.

1.  **Phase 1: STARTUP (시작과 초기화):** `main.tsx`에서 인증(OAuth, API key 등), 모델 선택, 설정 로드, Git 상태 및 `CLAUDE.md` 컨텍스트 수집을 수행한다.
2.  **Phase 2: QUERY LOOP (대화 루프):** `query.ts`가 핵심 엔진 역할을 하며 Claude API와 스트리밍 통신을 한다. 응답에서 `tool_use` 블록을 감지한다.
3.  **Phase 3: TOOL EXECUTION (도구 실행):** 입력 검증(Zod), 권한 확인(rules + classifier) 후 45개 이상의 내장 도구나 MCP 도구를 실행하고 결과를 API에 반환한다.
4.  **Phase 4: DISPLAY (결과 표시):** Ink TUI를 통해 메시지, 디프(diff), 진행 상황을 터미널에 렌더링한다.

핵심은 2단계와 3단계 사이의 루프다. 사용자의 한 번의 요청에도 내부적으로는 여러 번의 도구 실행과 API 호출이 반복되며 문제를 해결한다.

## 3. 실행 모드
*   **REPL Mode:** 일반적인 대화형 터미널 UI 모드.
*   **Headless Mode:** UI 없이 SDK나 파이프라인에서 호출되는 모드 (`QueryEngine` 클래스 활용).
*   **Coordinator Mode:** 리더 에이전트가 여러 워커 에이전트를 관리하는 멀티에이전트 모드.
*   **Bridge Mode:** 로컬 CLI와 클라우드의 claude.ai 웹 인터페이스를 연결하는 모드.
*   **Assistant (Kairos) / Daemon / Viewer Mode:** 상시 대기, 백그라운드 실행, 읽기 전용 관찰 모드 등.

## 4. 시작과 초기화 과정
*   **main.tsx:** 약 800KB의 거대한 단일 파일로, 시작 성능(디스크 I/O 최소화)을 위해 의도적으로 합쳐져 있다. 병렬 I/O 프리페치, 조건부 모듈 로딩, 인증 및 모델 확정 단계를 거친다.
*   **컨텍스트 수집:** Git 브랜치, 상태, 최근 커밋 등의 시스템 컨텍스트와 `CLAUDE.md` 등의 사용자 컨텍스트를 수집하여 모든 대화 턴에 주입한다.

## 5. 쿼리 루프 (Query Loop)
`query.ts`는 비동기 제너레이터 패턴을 사용하여 스트리밍 응답을 처리한다. 매 턴마다 메시지 전처리(압축/요약), API 호출, 에러 복구(토큰 한도 초과 시 에스컬레이션 등), 도구 실행, 후처리를 반복한다.

## 6. 도구(Tool) 시스템
*   **원리:** AI는 텍스트만 생성하므로, Claude Code가 파일 읽기/수정, 셸 명령 실행 등의 기능을 도구 형태로 제공한다.
*   **주요 도구:** `BashTool`(기본 거부 설계로 안전성 확보), `AgentTool`(서브에이전트 생성), `FileEditTool`(퍼지 매칭 기반 수정), `GrepTool` 등 45개 이상.
*   **실행 파이프라인:** 10단계(이름 조회, 중단 신호 확인, 입력 검증, 훅 실행, 권한 확인, 실제 실행, 결과 매핑, 대용량 결과 디스크 저장, 사후 훅, 텔레메트리 기록)를 거쳐 안전하게 실행된다.

## 7. 권한(Permission) 시스템
*   **모드:** Default(위험 작업 시 질문), Auto(AI 분류기가 위험도 평가 후 자동 승인), Plan(읽기 전용만 허용), Bypass(전부 승인) 모드가 존재한다.
*   **계층:** 설정 소스(Local > Project > User)와 규칙에 따라 동작하며, 위험한 셸 명령은 파서를 통해 구조적으로 분석하여 차단한다.

## 8. UI 및 상태 관리
*   **Ink TUI:** React 기반 프레임워크로 이중 버퍼링, 객체 풀링, 더티 추적 기술을 사용하여 터미널 환경에서 고성능 UI를 구현한다.
*   **Zustand:** 불변(Immutable) 글로벌 상태 관리를 통해 예측 가능한 상태 변경을 보장한다.

## 9. 고급 시스템
*   **Bridge & Remote:** 로컬 환경과 클라우드 세션을 연결하여 웹에서도 로컬 코드를 다룰 수 있게 한다.
*   **Coordinator:** 복잡한 작업을 위해 '조사 - 종합 - 구현 - 검증' 단계로 워커들을 가동하는 멀티에이전트 아키텍처.
*   **Memory:** 영속적인 마크다운 파일을 통해 사용자 역할이나 프로젝트 고유 정보를 세션 간에 유지한다.
*   **Auto-Compact:** 대화가 길어지면 토큰 한계에 맞춰 지능적으로 요약 및 압축을 수행한다.
